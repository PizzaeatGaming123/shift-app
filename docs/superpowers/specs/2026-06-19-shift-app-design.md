# シフト管理アプリ（デモ）設計書

- 対象顧客: ラーメン店「株式会社暁夢」（岡山県倉敷市・3店舗）
- 目的: 手書き＋Excel転記のシフト運用を、スマホ・タブレットで完結させるデモ
- スコープ: 機能1〜3（希望提出 / 店長の確認・作成 / 共有）。機能4（人件費）は今回対象外
- 最優先方針: 「ド素人でも簡単と思えるくらい簡単」。タップ中心、文字入力は最小限

## 1. アーキテクチャ

- **React + Vite + TypeScript**。`npm install` → `npm run dev` で起動。
- サーバー・認証なし。データは **localStorage** に保存（リロードしても保持＝「保存される」体験）。
- 状態管理: React Context + `useReducer` を1つ。全画面が同じストアを参照。
- レスポンシブ（スマホ／タブレット対応）。コーポレートカラーはオレンジ。

### 将来の拡張余地
本番運用時はバックエンド（例: Spring Boot ＋ DB）を後付けし、localStorage 層をAPI層に差し替える想定。Reactフロントは再利用可能。今回はその差し替えがしやすいよう、データ入出力を `store/` に集約する。

## 2. 画面構成

全画面共通のヘッダーで切り替える（ログイン認証なし）。

- **ヘッダー**: 左=「暁夢」ロゴ（オレンジ）、中央=店舗選択（中島店／新田店／早島店）、右=役割切替トグル（スタッフ用 / 店長用）。
- **対象月**の前月・翌月切替。月カレンダーが各画面の中心。
- スタッフ用は「機能1（希望提出）」と「機能3（共有閲覧）」、店長用は「機能2（確認・作成）」と「機能3（共有閲覧）」にアクセスできる。

## 3. 機能仕様

### 機能1: スタッフ希望提出（スタッフ用）
- 上部の小さなドロップダウンで「自分（サンプルスタッフ）」を選択。
- 月カレンダーで日付をタップ → その日の希望を選ぶ: **早番 / 遅番 / 両方 / 休み希望**。
  - 早番 = 7:00-16:00、遅番 = 15:00-24:00。
  - タップでトグル。文字入力なし。
- 色分け: 早番=オレンジ、遅番=濃色（青系など区別しやすい色）、休み=グレー。
- 「提出」で確定（内部的には選択時点で都度保存。提出ボタンは確認用フィードバック）。

### 機能2: 店長の確認・作成（店長用）
- **一覧マトリクス**: 縦=スタッフ、横=日付。各セルにそのスタッフの希望（早/遅/休）を色表示。誰がいつ希望したか一目で分かる。
- **割り当て**: 店長が「日×時間帯」に誰を入れるか確定/解除できる（希望セルをタップ）。
- **充足表示**: 各日の早番・遅番の割り当て人数を表示。少なすぎ/多すぎを色で警告（例: 早番0〜1人=赤、適正=緑）。しきい値は定数で持ち、後で調整可能。

### 機能3: 完成シフトの共有（全役割が閲覧可）
- 確定した割り当てを、スタッフが見やすい**月カレンダー表示**。
- 各日に「早番: ◯◯, △△ / 遅番: …」を表示。色分けは機能1と統一。
- 印刷しやすいレイアウトを意識。

## 4. データ構造

localStorage に店舗ごとに保存。

```ts
Store        { id: string; name: string }
Staff        { id: string; name: string; storeId: string; employmentType: '正社員' | 'パート' }
Slot         = 'early' | 'late' | 'off'
ShiftRequest { staffId: string; date: string /* YYYY-MM-DD */; slot: Exclude<Slot,'off'> | 'off' }
  // 「両方」は early と late の2レコード、「休み希望」は off レコードで表現
Assignment   { date: string; slot: 'early' | 'late'; staffIds: string[] }
```

- 初期データ: 各店に数名のサンプルスタッフ（正社員＋パート）を自動投入（`seed.ts`）。空なら投入、既存があれば尊重。

## 5. コンポーネント分割（1ファイル1責務）

- `Header` — ロゴ／店舗選択／役割切替
- `MonthCalendar` — 月カレンダー描画（希望提出・共有で再利用）
- `RequestEditor` — 機能1（希望の選択・保存）
- `ManagerMatrix` — 機能2（一覧マトリクス＋割り当て＋充足表示）
- `SharedView` — 機能3（共有カレンダー）
- `store/` — Context・reducer・localStorage 入出力（API差し替えの単一窓口）
- `seed.ts` — サンプルデータ投入
- `lib/date.ts` — 月・日付ユーティリティ

## 6. エラー処理・エッジケース

- localStorage が使えない場合はメモリ上で動作継続（デモを止めない）。
- 同一スタッフ・同一日・同一スロットの重複希望は冪等に扱う。
- 月またぎ・うるう年は日付ユーティリティで吸収。
- スタッフ未選択で希望提出しようとしたら、選択を促す軽い表示。

## 7. テスト方針

- ロジック（reducer / 日付ユーティリティ / 充足判定）はユニットテスト。
- 主要フロー（希望提出→店長割り当て→共有表示）の結合的な確認をコンポーネントテストで担保。
