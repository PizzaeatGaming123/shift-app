# 参照資料準拠 UI 再構築 設計

## 目的

暁夢向けシフト管理アプリの管理者画面・スタッフ画面を、`要件定義/` に格納された画像と PDF の画面構成に忠実に再構築する。

一般的な SaaS UI、既存実装の見た目、AI が補完した装飾はデザイン判断の根拠にしない。既存の API・データモデル・動作する業務ロジックは可能な限り再利用し、画面構造と操作導線を参照資料に合わせて作り直す。

## 参照資料の優先順位

資料間で表現が異なる場合は、次の順番で採用する。

1. `要件定義/上のバー.png`
2. `要件定義/シフト一覧.png`
3. `要件定義/別パターン.png`
4. `要件定義/大きさ.png`
5. `要件定義/スクリーンショット 2026-06-22 111638.png`
6. `要件定義/スクリーンショット 2026-06-22 111747.png`
7. `要件定義/300.pdf`
8. `要件定義/start_manual.pdf`
9. `要件定義/要件定義.md`

既存 UI は機能の所在を調べる目的でのみ参照し、外観の正解にはしない。

## デザイン原則

### 資料にない意匠を追加しない

- 絵文字を UI アイコンとして使わない。
- 資料にない大型カード、強い角丸、グラデーション、装飾的な影を使わない。
- 資料にない説明文、キャッチコピー、バッジを追加しない。
- ホバーやトランジションは、操作対象を識別するための軽い背景色・影・色変化に限定する。
- シフトチップ、ボタン、入力欄の角丸は資料に見える半径だけを使う。

### 画面密度

- 管理画面は資料と同様に、白背景・細い罫線・小さめの操作部品・高密度な表を基本とする。
- 店長が読みやすいように標準表示は十分な文字サイズと行高を確保する。
- 表示設定で「小・標準・大」を選択できるようにし、既定値は標準とする。
- シフト一覧内部に独立した縦スクロール領域を作らず、ページ全体で縦スクロールする。
- 日付列が画面幅を超える場合のみ横スクロールを許可する。

### 色

- 上部ナビゲーションは資料の明るい青を基準にする。
- ナビ項目は白い縦罫線で区切る。
- 背景は白、区切りは薄いグレー、主要操作は青。
- 早番・中番・遅番・休みの色は単体画像と `300.pdf` のチップ表現を基準にする。
- 未確定シフトは点線または淡い枠、確定シフトは実線または淡い塗りで区別する。
- 色だけに依存せず、ラベル・線種・状態表示を併用する。

### シフト一覧上部

`上のバー.png` と `別パターン.png` を基準に、次の順で配置する。

1. 店舗選択
2. ポジション選択
3. シフト確定ボタンと未確定件数
4. 印刷
5. シフトの種類
6. 日・週・半月・月の表示切替
7. 前期間・次期間
8. 表示期間
9. 提出期限
10. 概要設定または表示項目設定
11. 追加募集中件数
12. ヘルプ募集

操作部品は資料と同じ高さ・余白感に揃える。大きなセグメントコントロールやカード状のタブにはしない。

### シフト一覧操作行

`別パターン.png` の構造を採用する。

- ポジション名
- 上部固定
- 縮小／拡大
- 出勤者のみ
- シフトパターン
- 希望シフト
- タスク
- 勤務メモ
- 一括操作
- 過去コピー
- スタッフ並び替え

各チェック項目は実際の表示内容に連動する。一括操作と並び替えは独立したボタンとして配置する。

### 月・半月表示

`シフト一覧.png` と `大きさ.png` を基準にする。

- 左端にスタッフ名、月間労働時間、必要な警告と属性を表示する。
- 日付見出しは日付と曜日を縦に配置する。
- 土曜日は青、日曜日・祝日は赤を基調にする。
- スタッフセル内に、希望シフト、確定シフト、勤務時間、勤務メモを資料と同じ順序で表示する。
- 希望シフトと確定シフトは線種・背景・色で区別する。
- スタッフ行は資料の高さを基準にし、余計なプロフィールカード表現を使わない。
- スタッフ並び替えは資料と同じ位置に置き、標準・氏名・労働時間・ランク等で切り替える。

### 週表示

`別パターン.png` を基準にする。

- 1 週間分の日付列を表示する。
- スタッフごとのシフトを時刻範囲で表示する。
- 休み、希望、確定、メモを別レイヤーで表示する。
- 表示項目のチェック状態を即時反映する。

### 日表示・バー表示

`300.pdf` のバー表示を基準にする。

- 横軸を営業時間、縦軸をスタッフとする。
- 勤務開始・終了時刻をバーの端で調整できる。
- 固定シフトと確定シフトを区別する。
- モデルシフトの必要人数と実配置人数を時間帯ごとに表示する。

### 集計行

表示項目として次を追加・削除できるようにする。

- 売上計画
- 人時売上高
- 総労働時間
- 人件費と人件費率
- モデルシフト
- ランク計
- 店舗メモ
- ポジションメモ

集計行はまとめて表示・非表示にできるほか、個別にも表示・非表示を選べる。

## 調整

### 追加募集

`300.pdf` と `start_manual.pdf` を基準に、日付・時間帯・募集人数・メッセージを入力して募集する。募集中件数はシフト一覧上部に表示する。

### チャット

- 左側にスタッフ一覧
- 右側に選択スタッフとの会話
- 未読表示
- 送信時刻
- 業務連絡用途の簡潔な入力欄

### 他事業所ヘルプ

- 期間
- 希望勤務先
- ポジション
- スタッフ
- 元店舗
- 移動先店舗
- 割り当て操作

## シフト確定

`start_manual.pdf` の STEP5 を基準にする。

- 確定対象のポジションと日付を選択する。
- 未確定が残る場合は対象と件数を表示する。
- 確定後は対象スタッフへ通知する。
- 確定後の再調整では、変更対象スタッフのみ通知対象にする。
- 確定シフト画面では日・週・月を切り替えられる。

## スタッフ・管理者登録

`start_manual.pdf` の STEP1 と `300.pdf` の登録画面を基準にする。

### スタッフ登録

- 雇用形態
- 招待 URL

- 登録状況

### 管理者登録

- 権限
- 氏名
- 生年月日

## データと API

- 現在の認証、店舗、スタッフ、希望、割り当て、メモ、追加募集 API は再利用する。
- シフトパターン、固定シフト、表示色、回収設定、通知、チャット、他事業所ヘルプ、確定履歴はバックエンドに永続化する。
- `localStorage` は画面表示設定など端末単位の設定に限定する。
- 店舗・スタッフ・権限・シフトに関わる業務データはバックエンドを正とする。
- 店長専用更新 API は MANAGER 権限で保護する。

## エラー処理

- 保存失敗時は画面を閉じず、操作対象の近くに原因を表示する。
- 入力値はフロントとバックエンドの両方で検証する。
- 日付範囲、時刻順序、文字数、人数、権限を検証する。
- API エラーを HTTP ステータスだけで表示せず、日本語の利用者向けメッセージに変換する。

## アクセシビリティ

- キーボードでナビ、シフトセル、ダイアログを操作可能にする。
- フォーカスリングを維持する。
- チェック状態、未確定状態、色設定は ARIA と文字でも伝える。
- `prefers-reduced-motion` ではホバー移動やアニメーションを抑制する。

## 完了条件

- 参照資料に存在しない主要な UI 表現が残っていない。
- 単体画像の上部ナビ・シフト一覧・表示切替と、実画面の構造・色・密度が一致している。
- PDF に掲載された主要機能が、資料と同じ導線で操作できる。
- スタッフと管理者の権限が分離されている。
- 業務データが再読み込み後も保持される。
- 自動テスト、型チェック、ビルドが成功する。
- 視覚比較で重大な差分が残っていない。
