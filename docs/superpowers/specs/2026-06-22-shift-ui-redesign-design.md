# 暁夢シフト UI 全面刷新 設計（Design）

**日付:** 2026-06-22
**対象:** `frontend/`（React + Vite + TypeScript）
**ゴール:** 既存の3機能（希望提出・割り当て・確定シフト共有）はそのままに、見た目を「らくしふ」級の商用 SaaS 品質へ全面刷新する。清潔＆プロ（インディゴ基調）・レスポンシブ・使っていて気持ちのいい操作感。

## スコープ

- **やること:** 見た目（CSS）の全面刷新、UI 補助コンポーネントの追加、操作感（マイクロ操作・ローディング・空状態・トースト）の改善。
- **やらないこと:** 機能追加（勤怠の打刻など）、バックエンド/API の変更、ドメインロジック（`lib/date`・`store/requests`・`store/assignments`）の変更。

## 方針（採用アプローチ）

**A: 手書きモダンCSS刷新（依存追加なし）。** `styles.css` をデザイントークン（CSS 変数）ベースに全面リライトし、React 構造は概ね維持。Tailwind/UI ライブラリは導入しない（オリジナル性・軽さ・既存コードとの相性を優先）。

## デザインシステム（トークン）

CSS 変数で一元管理し、全画面で一貫させる。

- **配色**
  - プライマリ: `#4f46e5`（濃 `#4338ca`）
  - 背景: ページ `#f8fafc` / カード `#ffffff`
  - 文字: 主 `#0f172a` / 副 `#64748b` / 罫線 `#e2e8f0`
  - 早番: アンバー `#f59e0b` / 遅番: インディゴ `#6366f1` / 休: スレート `#94a3b8`
  - 充足: 不足 `#ef4444` / 適正 `#10b981` / 過多 `#f59e0b`
- **角丸:** 8 / 12 / 16px
- **影:** 2段階（sm・md）でふわっと
- **余白:** 4 の倍数で統一
- **文字:** 日本語UIフォント。見出し 700・本文 400。数字は等幅で表を読みやすく
- **動き:** 150〜200ms のスッとした遷移。タップで軽く沈む。保存時にトースト

## 共通レイアウト（アプリの骨格）

```
┌──────────────────────────────────────────┐
│ 🍜 暁夢シフト   [店舗▾]        山田 店長 ▾ │ ← トップバー（sticky）
├──────────────────────────────────────────┤
│        ‹  2026年 7月  ›   [今月]           │ ← 月ナビ
├──────────────────────────────────────────┤
│  [ 希望確認・割り当て ]  [ 確定シフト ]      │ ← セグメントタブ
│              （各画面の中身）               │
└──────────────────────────────────────────┘
   スマホ時：タブは下部に固定（親指で届く）
```

- トップバー: ロゴ／店舗セレクト／ユーザーチップ（名前＋役割バッジ、押すとログアウト）。
- 月ナビ: 前/次月の矢印＋「今月」ショートカット。
- タブ: PC はインライン、スマホは下部固定。

## 画面ごとの設計

### ログイン（`Login.tsx`）
中央カード。ブランドロゴ＋キャッチコピー。デモアカウントを **ワンタップ自動入力チップ** で提示し「お試しログイン」を快適に。エラーは赤で控えめに表示。

### スタッフ：希望提出（`RequestEditor.tsx`）
- 上部に **サマリーバー**（例: 「今月の提出 12/30日 ・ 早番5 遅番4 休3」）。
- カレンダーは色チップ表示。日付タップで **ボトムシート（スマホ）/ インラインパネル（PC）**。早番/遅番/早番+遅番/休み/クリアを大きなボタンで。
- 保存時にトースト「保存しました ✓」。

### 店長：割り当てマトリクス（`ManagerMatrix.tsx`）— 主役
- 1行目（日付）と1列目（スタッフ名）を **固定**（sticky）。横スクロールでズレない。
- 希望セルは色分け（早=アンバー/遅=インディゴ/休=グレー）。タップで割り当て ON/OFF、割り当て中はリング＋太字。
- 最下部に **充足インジケータ**（早番/遅番の人数を数字＋色＋小さなバー）。
- 上部に **凡例**（早/遅/休/不足…）を常設。

### 確定シフト共有（`SharedView.tsx`）
読みやすさ最優先のカレンダー。各日に「🌅早番 ◯◯、△△ / 🌙遅番 □□」をチップ＋名前で。出勤者ゼロの日は控えめ表示。

## 追加する補助コンポーネント

各々 1 つの責務に絞った小さな部品として追加（`frontend/src/components/ui/` 配下を想定）:

- `Toast` … 保存などの通知（自動で消える）
- `Skeleton` … データ取得中のプレースホルダ
- `Badge` … 役割バッジ・充足バッジ
- `BottomSheet` … スマホでの希望選択パネル（PC ではインライン表示に切替）
- `SummaryBar` … スタッフ画面の提出サマリー
- `Legend` … マトリクスの凡例

## 仕上げ（品質ポイント）

- **ローディング:** 取得中はスケルトン（レイアウトをガタつかせない）。
- **空状態:** 希望ゼロ/割り当てゼロに優しいメッセージとアイコン。
- **マイクロ操作:** タブ・ボタン・チップに軽いトランジション、タップフィードバック。
- **アクセシビリティ:** コントラスト確保、`aria-label` 維持、キーボード操作可。
- **レスポンシブ:** スマホ＝1カラム＋下部タブ、PC＝広い表。

## 既存コードへの影響・互換性

- `styles.css` を全面リライト（トークン化）。`App.tsx`/各コンポーネントは見た目のためのマークアップ調整中心で、API 連携（`AppContext`・`api/client`）とロジックは変更しない。
- テキストラベル（例: 「早番人数」「遅番人数」）と既存の意味的構造は維持するため、**既存テスト（date/requests/assignments/client/ManagerMatrix）はそのまま通す**ことを前提とする。テストが壊れる変更が必要なら、テストも併せて更新する。

## テスト/検証

- `npx tsc --noEmit` がエラーなし。
- `npm test` が全て green。
- 手動: スマホ幅/PC 幅で各画面の崩れがないこと、ログイン→希望提出→割り当て→確定シフトの一連が動くこと。

## 既知の非対象（YAGNI）

- ダークモード、勤怠打刻、複数店舗集約ビュー、人件費（機能4）は対象外。
