# シフト管理アプリ システム概要（現状）

株式会社暁夢（ラーメン店3店舗）向けのシフト管理デモ。クラウドサービス「らくしふ」の店長向け画面を参考に、希望提出 → 作成（割り当て）→ 確定共有までを Web アプリで再現する。

## 構成

- フロントエンド: React + Vite + TypeScript（`frontend/`）。状態は `store/AppContext.tsx`、API は `api/client.ts`。
- バックエンド: Spring Boot + Spring Security + JPA + H2（ファイルDB）（`backend/`）。
- 認証: セッションベース。ログイン/ログアウト/me。デモアカウントのパスワードは全員 `password`。
- データ: 店舗・スタッフ・希望・割り当て・各種メモはバックエンドに永続化。表示設定や売上計画など店長の好みはブラウザ（localStorage）に保持。

## 登場ロール

- 店長（MANAGER）: マトリクスで割り当て・確定・各種設定・スタッフ管理。全機能。
- スタッフ（STAFF）: 希望シフトの提出（早番/中番/遅番/休み＋ひとことメモ）。

## 主な機能（利用の流れ）

このアプリは「スタッフが希望を出す → 店長が割り当てる → 確定して共有する」という流れで使う。各ステップで何ができるかを示す。

### ステップ1：スタッフが希望を出す
- 月カレンダーで日付をタップし、その日の希望を **早番・中番・遅番・休み** から選ぶだけ。
- 「変わってほしい」「応援できます」などの **ひとことメモ** を添えられる。
- 自分の提出状況（提出した日数・内訳）がひと目で分かる。

### ステップ2：店長が割り当てる（シフト一覧）
- 全スタッフの希望が **1つの表（カレンダー）** に集まる。誰がいつ希望を出したかが一覧できる。
- 各セルをクリックすると、そのスタッフをその日に **割り当て / 解除**。
- **「一括操作」** で、希望どおりの未割り当てをまとめて割り当て。
- 表の上部に **その日の状況** が並ぶ：必要人数の **過不足（不足は赤・適正は緑）**、総労働時間、人件費の目安、売上計画。
- 見え方を切り替えられる：**文字サイズ（小/標準/大）**、集計行の表示オン/オフ、表示範囲（日/週/半月/月）、区分の絞り込み、スタッフの並び替え（時間順/ランク順/氏名順）。
- まだ希望を出していないスタッフは **「未収アラート」** で分かる。急な欠員には **「追加募集」** をその日に出せる。

### ステップ3：確定して共有する
- **「シフト確定」** ボタンで月を確定（確定済みの表示に変わる）。
- 確定したシフトを「各日の出勤者」として表示・**印刷**・**CSV出力** できる。

### 経営の見える化
- **人件費**：その日の割り当てから人件費と、売上計画に対する **人件費率（%）** を自動計算。
- **人時売上高**・**総労働時間** など、経営判断に使う数値を集計表示。
- **労務チェック**：連続勤務や労働時間が多いスタッフに **注意バッジ** を表示。

### スタッフ・店舗の管理
- **スタッフ登録／管理者登録**、**ランク（労働力）・スキル** の設定。
- **店舗・部門・権限**（スタッフに許可する操作）の設定。
- 営業時間・シフト回収（提出期限・通知）・通知・外部連携などの **各種設定**。

## データモデル（バックエンド）

- `Store`、`Staff`(雇用形態/ロール/ランク/スキル)、`ShiftRequest`(early/mid/late/off)、`ShiftAssignment`(early/mid/late)、`DayNote`(スタッフ日次メモ)、`StoreNote`(店舗メモ)、`Recruitment`(追加募集)。
- 主なAPI: 認証、店舗・スタッフ取得/作成、スタッフ更新、希望の取得/設定、割り当ての取得/付与/解除、各メモ・追加募集の取得/更新。書き込み系の一部は店長権限のみ。

## デモの起動

- バックエンド: `cd backend && ./mvnw.cmd spring-boot:run`（8080）。フロント: `cd frontend && npm run dev`（5173、`/api` を8080へプロキシ）。
- ログイン: 画面の「お試しログイン」チップ（例: 中島店 店長 `nakashima-mgr` / スタッフ `nakashima-1`）。
